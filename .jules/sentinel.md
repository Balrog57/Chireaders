## 2025-06-03 - [Race Condition in AsyncStorage Background Tasks]
**Vulnerability:** A "Check-Then-Act" race condition was found in `BackgroundNotificationTask.js`. The task read data from `AsyncStorage`, performed long-running network operations, and then overwrote the storage with the stale initial data plus updates. This caused user changes (e.g., deleting a favorite) made during the task execution to be lost.
**Learning:** Background tasks in React Native headless JS operate independently of the UI state and contexts. Direct `AsyncStorage` usage without re-validation is dangerous for data integrity when user interaction is possible concurrently.
**Prevention:** Always implement a "Fetch-Merge-Save" strategy for background tasks: collect updates first, then re-read the storage source of truth immediately before saving, merging the updates into the fresh state.
## 2025-02-23 - WebView message origin validation
**Vulnerability:** WebView component was accepting messages from injected JavaScript without validating the origin URL, leading to potential Cross-Site Scripting (XSS) or malicious data injection if the user navigates to an untrusted domain or if the target domain is compromised.
**Learning:** Even if `initialUrl` is controlled, WebViews can navigate elsewhere (e.g., via user clicks, redirects, or compromised ads). The `onMessage` handler receives messages from the currently loaded frame, which might be malicious. Therefore, any message payload processed natively must first be gated by an origin validation check.
**Prevention:** Always validate `event.nativeEvent.url` against an allowed list of domains (e.g., using `isValidChiReadsUrl`) inside the `onMessage` handler of a `WebView` before parsing and processing the message data.
