## 2024-05-24 - Missing Origin Validation in WebView Message Handler
**Vulnerability:** The `handleMessage` function in `src/screens/BrowserScreen.js` receives messages from the WebView using `onMessage`, but it does not validate the origin (URL) of the sender before processing the message.
**Learning:** This is a common React Native vulnerability where a WebView navigated to a malicious domain can execute injected JavaScript that sends messages to the native app. While the current message types (`styleApplied`, `titleChanged`) only update state and don't seem immediately critical, an attacker could potentially spoof the title or prepare for more complex attacks if more powerful message handlers are added later. The app already has a `isValidChiReadsUrl` utility designed exactly for this purpose.
**Prevention:** Always validate `event.nativeEvent.url` in WebView `onMessage` handlers against an allowlist of trusted domains before parsing or acting on the message payload.
